# 기능 1~3 학습 기록

## 오늘 학습한 내용
* 2주차 과제의 기능 1부터 기능 3까지 진행했다.
* 기술 문서의 텍스트를 검색에 활용할 수 있도록 소문자 변환, 특수문자 제거, 중복 공백 정리 등의 전처리 과정을 구현했다.
* 두 벡터의 내적과 노름을 이용해 코사인 유사도를 NumPy로 직접 계산하는 함수를 구현했고, 같은 벡터·직교 벡터·영벡터를 이용해 동작을 확인했다.
* 또한 질문과 문서에서 겹치는 단어의 개수를 점수로 사용하는 키워드 Baseline 검색을 구현했다.

## 구현한 기능
* `content` 결측치 제거
* `preprocess()` 텍스트 전처리 함수 구현
* `content_clean` 컬럼 생성
* `cosine_similarity_numpy()` 직접 구현
* 벡터 크기 불일치 예외 처리
* 영벡터 입력 시 0으로 나누는 문제 방지
* `keyword_search()` 키워드 검색 구현
* 질문과 문서의 단어 교집합을 이용한 점수 계산
* 점수 기준 Top-3 문서 반환

## 실행 결과
* 같은 벡터의 코사인 유사도: 1.0000
* 직교 벡터의 코사인 유사도: 0.0000
* 영벡터가 포함된 경우: 0.0000
* Keyword Baseline 검색에서 Top-3 결과가 정상적으로 출력됨

## 막힌 부분
* 코사인 유사도를 계산할 때 벡터의 크기가 0이면 분모가 0이 될 수 있다는 점과, 키워드 검색에서 질문과 문서를 어떤 방식으로 비교해야 하는지가 처음에는 헷갈렸다.

## 해결 과정
* 두 벡터의 노름을 먼저 계산하고 하나라도 0이면 유사도 0.0을 반환하도록 처리했다.
* 키워드 검색에서는 질문과 문서를 각각 단어 집합으로 변환하고, 두 집합의 교집합 크기를 검색 점수로 사용했다.

# 기능 4~6 학습 기록

## 오늘 학습한 내용
* `TfidfVectorizer`를 이용해 전처리된 기술 문서를 TF-IDF 벡터 행렬로 변환했다.
* TF-IDF가 여러 문서에 흔하게 등장하는 단어의 가중치는 낮추고, 특정 문서를 구별하는 단어의 가중치는 높이는 방식이라는 것을 학습했다.
* 문서 벡터화에 사용한 vectorizer를 재사용해 질문을 동일한 TF-IDF 공간의 벡터로 변환했다.
* 질문 벡터와 각 문서 벡터 사이의 코사인 유사도를 계산하고, 유사도가 높은 Top-3 문서를 반환하는 검색 기능을 구현했다.
* Keyword Baseline과 TF-IDF 검색을 동일한 질문으로 실행해 두 검색 결과의 차이를 비교했다.

## 구현한 기능
* `TfidfVectorizer` import 추가
* `build_tfidf()` TF-IDF 문서 벡터화 함수 구현
* `max_features=5000`, `min_df=2`, `stop_words="english"` 옵션 적용
* TF-IDF 행렬 크기와 사용된 단어 수 출력
* TF-IDF 행렬과 학습된 vectorizer 반환
* `tfidf_search()` TF-IDF 기반 검색 함수 구현
* 기존 vectorizer를 이용한 질문 벡터 변환
* 직접 구현한 `cosine_similarity_numpy()`를 이용한 전체 문서 유사도 계산
* 코사인 유사도 기준 Top-3 문서 반환
* 기능 1부터 기능 5까지 `main()`에서 순서대로 연결
* Keyword Baseline과 TF-IDF 검색 결과 동시 출력
* 두 검색 방식의 차이에 대한 코드 주석 작성

## 실행 결과
* TF-IDF 행렬 크기: `(실제 실행 결과 입력)`
* TF-IDF에 사용된 단어 수: `(실제 실행 결과 입력)`
* TF-IDF 검색 Top-1 문서: `(실제 실행 결과 입력)`
* Keyword Baseline과 TF-IDF 검색에서 각각 Top-3 문서가 정상적으로 출력됨
* `python list comprehension` 질문에서 `D001 Python List Comprehension` 문서가 TF-IDF 검색 최상위에 나타나는 것을 확인함

## 막힌 부분
* 질문을 문서와 동일한 TF-IDF 벡터 공간으로 변환하는 방법과, 희소 행렬에서 각 문서 벡터를 꺼내 코사인 유사도 함수에 전달하는 과정이 헷갈렸다.
* Keyword Baseline과 TF-IDF 검색 결과가 서로 다른 이유를 처음에는 명확하게 이해하기 어려웠다.

## 해결 과정
* 문서 벡터화에 사용한 `TfidfVectorizer`를 질문에도 그대로 재사용하고, `transform()`을 사용해 질문을 문서와 동일한 차원의 벡터로 변환했다.
* 질문과 문서의 희소 행렬을 `toarray()[0]`으로 1차원 NumPy 배열로 바꾼 뒤, 직접 구현한 `cosine_similarity_numpy()`에 전달했다.
* Keyword Baseline은 겹치는 단어의 개수만 계산하기 때문에 일반적인 단어가 포함된 무관한 문서도 높은 점수를 받을 수 있지만, TF-IDF는 흔한 단어의 가중치를 낮춰 문서를 구별하는 핵심 단어가 검색 결과에 더 크게 반영된다는 점을 확인했다.
