# Week 01 학습 기록

## 오늘 학습한 내용
* Python 가상환경을 생성하고 프로젝트별로 패키지와 실행 환경을 관리하는 방법을 실습했다.
* pandas의 `read_csv()`를 사용해 CSV 파일을 DataFrame으로 불러오는 방법을 학습했다.
* DataFrame의 `shape`, `columns`, `dtypes`, `head()`를 이용해 데이터의 행·열 수, 컬럼명, 자료형, 상위 데이터를 확인했다.
* `value_counts()`와 반복문을 이용해 카테고리별 문서 수와 비율을 분석했다.
* 문자열의 `split()`을 활용해 문서별 단어 수를 계산하고, 카테고리별 평균 문서 길이를 구했다.
* `isnull()`을 이용해 컬럼별 결측치 수와 비율을 확인하고, 결측치 비율에 따라 심각도를 구분했다.
* NumPy를 사용해 문서 길이의 평균, 표준편차, 중앙값, 최솟값, 최댓값을 계산했다.
* NumPy의 조건 필터링을 이용해 50단어 미만의 짧은 문서를 탐색했다.
* NumPy와 pandas의 표준편차 계산 결과를 일치시키려면 `ddof=1`을 사용해야 한다는 점을 확인했다.
* pandas 버전에 따라 문자열 컬럼의 자료형 표시가 달라질 수 있다는 점을 학습했다.
* pandas 3.0부터 문자열 컬럼이 기본적으로 `object`가 아닌 `str` 자료형으로 추론되며, 현재 실행 환경에서도 모든 문자열 컬럼이 `str`로 출력되는 것을 확인했다.
* 이번 데이터 탐색이 이후 구현할 문서 벡터화와 검색 시스템의 입력 데이터 품질을 검증하는 단계라는 점을 이해했다.

## 오늘 구현한 기능
* GitHub 저장소와 `data`, `week01`, `learning-log` 중심의 프로젝트 구조를 생성했다.
* `load_data()` 함수를 구현해 CSV 파일의 존재 여부를 확인하고 데이터를 불러오도록 했다.
* 파일 경로, 문자 인코딩, 빈 파일, CSV 형식 오류에 대한 예외 처리를 추가했다.
* `explore_structure()` 함수를 구현해 데이터의 행·열 수, 컬럼명, 자료형, 상위 5개 행을 출력하도록 했다.
* `show_category_distribution()` 함수를 구현해 카테고리별 문서 수, 비율, 평균 단어 수를 계산하도록 했다.
* `check_missing()` 함수를 구현해 컬럼별 결측치 수, 비율, 심각도를 출력하도록 했다.
* `numpy_doc_stats()` 함수를 구현해 문서 길이의 주요 통계값을 계산하도록 했다.
* NumPy 조건 필터링을 사용해 50단어 미만 문서의 ID, 제목, 단어 수를 확인하도록 했다.
* pandas의 `describe()` 결과와 NumPy로 직접 계산한 통계값을 비교하도록 했다.
* 각 기능을 함수로 분리하고 `main()`에서 순서대로 실행되도록 연결했다.

## 막힌 부분
* 과제 안내에서는 문자열 컬럼의 자료형이 `object`로 제시되었지만, 실제 실행 결과에서는 모든 컬럼이 `str`로 출력되어 원인을 확인할 필요가 있었다.
* NumPy와 pandas에서 계산한 표준편차 값이 서로 조금 다르게 출력되었다.
* 결측치를 제거한 `content` 데이터의 단어 수와 원본 데이터의 `doc_id`, `title`을 다시 연결하는 과정이 헷갈렸다.

## 해결 과정
* 자료형 차이는 Python 버전 때문이 아니라 pandas 버전과 문자열 자료형 추론 방식의 변화 때문이라는 점을 확인했다.
* pandas 3.0부터 문자열 데이터가 기본적으로 `str` 자료형으로 추론되므로, 현재 환경에서 `doc_id`, `title`, `category`, `content`, `source`가 모두 `str`로 출력되는 것은 정상적인 결과임을 확인했다.
* NumPy의 `np.std()`는 기본적으로 모집단 표준편차를 계산하지만, pandas의 `std()`와 `describe()`는 표본표준편차를 계산한다는 차이를 확인했다.
* `np.std(document_lengths, ddof=1)`로 수정해 pandas와 동일한 표본표준편차 계산 기준을 적용했다.
* `dropna()`를 실행해도 원본 DataFrame의 인덱스가 유지된다는 점을 이용했다.
* 결측치가 제거된 `content`의 인덱스로 원본 DataFrame의 `doc_id`와 `title`을 조회한 뒤, NumPy에서 계산한 단어 수를 연결했다.

## 학습 소감
이번 과제는 단순히 CSV 파일을 읽고 출력하는 작업이 아니라, 검색 시스템을 구현하기 전에 데이터의 구조와 품질을 점검하는 과정이었다. 카테고리별 데이터 수가 지나치게 차이 나거나 문서 길이가 너무 짧으면 이후 벡터화와 검색 결과에도 영향을 줄 수 있다는 점을 이해했다. 또한 같은 코드라도 라이브러리 버전에 따라 출력되는 자료형이 달라질 수 있다는 점을 직접 확인했다. 예시 결과와 실제 결과가 다를 때 코드가 잘못되었다고 바로 판단하기보다, 사용 중인 라이브러리 버전과 공식 변경 사항을 확인하는 과정이 필요하다는 점을 배웠다. pandas는 데이터 구조 탐색과 집계에 편리하고, NumPy는 수치 통계와 조건 필터링에 적합하다는 차이도 직접 비교할 수 있었다. 앞으로 문서를 벡터화하고 검색 기능을 구현할 때도 입력 데이터의 상태와 실행 환경을 먼저 검증하는 습관이 중요하다고 느꼈다.
